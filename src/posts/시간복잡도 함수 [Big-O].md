## 시간복잡도

- 간략하게 정리하자면 이 **알고리즘이 어느 정도의 성능을 가졌는지 설명하는 것**이다.
  어떤 문제를 해결하기 위해 짜여진 알고리즘이 얼마나 오래 걸릴지 에 대해 알아보기 위해서 쓴다고 보면 이해하기 쉬울 것이다.
- 좀 더 자세하게 설명하자면 시간복잡도는 **알고리즘을 수행하기 위해 프로세스(컴퓨터)가 수행해야 하는 연산을 수치화 한 것**이라고 한다.
  사실 알고리즘이 얼마나 오래 걸릴지는 단순히 코드에 따른 것만이 아니라 프로세스의 사항, 사용하는 프로그래밍 언어, 컴파일러(혹은 인터프리터)의 버전이나 그 개발자(?)에 따라 천차만별이기 때문에 프로세스가 수행해야 하는 연산의 양을 수치 화해서 어느 정도 걸릴 지를 대강 예측하는 것 이다.
  - 시공간복잡도를 표기하기 위해서는 다양한 표기법을 사용할 수 있다.
    하지만 여기에서는 시간복잡도에 대해서만 다룰 예정이기에 시간복잡도를 표기하는 표기법만 기술할 예정이다. (나중에 공간복잡도 정리할 때 다른 거 쓸 예정, 애초에 공간복잡도는 잘 몰?루)

## 점근적 표기법

- 시간복잡도를 표기하기 위한 표기법에는 **점근적 표기법**이 있다.
  점근적 표기법의 점근적(Asymptotic)의 뜻은 직역하자면 점점 가까워지는 이란 뜻으로 가장 큰 영향만 계산한다는 의미로 해석할 수 있다. (결과 도출에 가까운 요소만 생각한다고 보면 됨)
- 점근적 표기법은 3가지로 나눌 수 있는데, 그 중 하나가 이번에 설명할 가장 많이 사용하는 표기법인 **Big-O 표기법**이다.

### Big-O 표기법

- Big-O, 말 그대로 큰O(대문자 O)를 사용하는 이 표기법은 **변수n에 따라 실행되는 명령의 수를 나타내는 함수**이다. (시간복잡도 기준)
  점근적 표기법의 일종인 만큼 불필요한 연산은 전부 제거하고 영향이 큰 연산만 계산하며, 값이 클 수록 명령의 수가 많아 시간이 오래 걸린다는 뜻이다.
- 아래는 Big-O의 복잡도를 나타내는 표이다.

![../img/Big-O.png](../img/Big-O.png)

- 가장 영향을 많이 주는 변수 N의 차수(이렇게 표현하는 게 쉬울 것 같음)가 증가함에 따라 복잡도가 증가하게 되는 것을 확인할 수 있다.
- 아래 표를 참고하여 대표적인 Big-O 표기들의 시간복잡도와 특징들을 확인해 보자.

| 표기         | 시간복잡도                             | 특징                                                                  |
| ------------ | -------------------------------------- | --------------------------------------------------------------------- |
| O(1)         | 상수 시간                              | 문제를 해결하는데 오직 한 단계만 처리함.                              |
| O(log₂ n)    | 로그 시간                              | 문제를 해결하는데 필요한 단계들이 연산마다 특정 요인에 의해 줄어듬.   |
| O(n)         | 선형 시간                              | 문제를 해결하기 위한 단계의 수와 입력값 n이 1:1 관계를 가짐.          |
| O(n\*log₂ n) | 선형 로그 시간                         | 문제를 해결하기 위한 단계의 수가 n\*(log₂n) 번만큼의 수행시간을 가짐. |
| O(n^2)       | 2차 시간                               | 문제를 해결하기 위한 단계의 수는 입력값 n의 제곱.                     |
| O(C^n)       | 지수 시간                              | 문제를 해결하기 위한 단계의 수는 주어진 상수값 C 의 n 제곱.           |
| O(n!)        | (나와있는 문서가 없음, 팩토리얼 시간?) | 문제를 해결하기 위한 단계의 수는 입력값 n의 팩토리얼 값               |

O(1) : 상수 시간

- 대표적인 Big-O 표기들의 예시 코드를 Python으로 작성하여 확인해 보자.(일부 표기만 기제)

```python
#헬로우 월드 예제
def main():
	print("Hello, World!")
```

O(n) : 선형 시간

```python
#리스트 원소 출력 예제
def main(list):
	for i in list:
    print(i)
```

O(n^2) : 2차 시간

```python
#구구단 예제
def main():
	for i in range(1,10):
		for j in range(1,10):
			print(f"{i} X {j} = {i*j}"
```

- 이런 시간복잡도를 구하는 가장 쉬운 방법은 반복문의 중첩 여부를 확인하는 것이다.
  반복문이 없다면 → O(1)
  반복문이 중첩되지 않다면 → O(n)
  반복문이 중첩되었다면 → O(n^2)
