## 문제 설명

- 문자열과 리스트를 주면, 리스트대로 문자를 재배열하는 문제라고 한다.
  자세한 설명을 위해 프로그래머스에서 설명을 복붙하자.

> 문자열 `my_string`과 이차원 정수 배열 `queries`가 매개변수로 주어집니다. `queries`의 원소는 [s, e] 형태로, `my_string`의 인덱스 s부터 인덱스 e까지를 뒤집으라는 의미입니다. `my_string`에 `queries`의 명령을 순서대로 처리한 후의 문자열을 return 하는 solution 함수를 작성해 주세요.

- 그렇다고 한다

## 접근

- ‘인덱스s부터 인덱스 e까지를 뒤집어라’ 라는 문구를 보고, 리스트에 함수 `reverse` 가 생각났다.
  리스트의 배열을 뒤집을 수 있는 아주 좋은 함수이다.
  이 함수를 응용하여 문자열을 리스트로 변환, 그리고 슬라이싱하고 `reverse`, 그리고 리스트를 다시 문자열로 변환하는 아이디어를 떠올려냈다.

```python
def solution(my_string, queries):
    for i in queries:
        my_list = list(my_string)
        my_list[i[0]:i[1]+1].reverse()
        my_string = ''.join(my_list)
    return my_string
```

- 이게 처음에 생각했던 코드이다. 이론상으로는 흠집이 없어 보인다. `queries` 의 인덱스 수 만큼 반복하고, 문자열을 리스트로 바꾸고, 슬라이싱, `reverse` 그리고 `join` 함수를 이용해 리스트를 문자열로 변경했다.
- 하지만 리턴값은 처음 주어진 문자열을 그대로 반환했다.
  도저히 수정할 방법이 생각나지 않아서 코파일럿한테 풀이를 물어봤다.

```python
def solution(my_string, queries):
    my_list = list(my_string)
    for i in queries:
        my_list[i[0]:i[1]+1] = my_list[i[0]:i[1]+1][::-1]
    return ''.join(my_list)
```

- 겁나게 간단하다.
  이 코드에서 궁금했던 것은 `reverse` 함수가 아닌 `[::-1]` 이라는 뭔 해괴망측한 문법을 사용하는 것이다.
  그래서 이것도 코파일럿한테 물어봤다.

## 알게 된 점

- 슬라이싱한 리스트에 직접적으로 뒤집으려고 시도하면 뒤집히지 않는다.
  이는 `reverse` 함수를 써도, `[::-1]` 문법을 써도 같은데, `reverse` 함수는 리스트를 직접적으로 바꾸려 하기 때문에 슬라이싱한 리스트를 변수에 따로 저장해야 하고, `[::-1]` 문법은 뒤집은 리스트를 반환하기 때문에 반환된 리스트를 그 위치에 다시 저장해 주어야 한다.
