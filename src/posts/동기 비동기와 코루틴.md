# 동기/비동기와 코루틴

## 동기

- 동기 의 사전적 의미는 **동시에 일어난다** 라는 뜻이다.
  프로그래밍에서의 동기는 **코드가 순차적으로 일어나는 것**을 의미한다.
  일반적인 프로그래밍은 전부 순차적 구조이기에 일반적으로 모든 프로그램은 동기이다.
- 동기적으로 작성된 프로그램은 한 작업이 실행되면 다른 작업이 실행될 수 없으며, 실행 중인 작업이 완료되고 값을 반환 받아야 다른 작업을 실행할 수 있다.

## 비동기

- 비동기 의 사전적 의미는 **동시에 일어나지 않는다** 라는 뜻이다.
  여기서도 알 수 있듯 앞서 설명한 동기와는 완전히 반대되는 개념으로, **코드가 순차적으로 일어나지 않을 수 있다.**
- 비동기적으로 작성된 프로그램은 한 작업이 실행 중이라 해도, 다른 작업이 호출되면 그 작업을 실행한다.
  여기서 실행되는 다른 작업은 실행 중인 작업을 종료 시키지 않고, 다른 두 작업이 같이 실행된다.

## 코루틴

- 코루틴은 비동기 처리를 위한 프로그래밍 구조이며 "함께(cooperative) 루틴" 이란 뜻이다.
  여기서 **루틴** 은 말 그대로 **계속 반복하는 것**으로, **`while` `for`**같은 반복문이라고 보면 좋을 듯 하다.
  코루틴의 개념에서 동기와 비동기를 다시 비교해보자.

- 위 그림 처럼 메인 루틴에서 서브 루틴을 요청하면, 서브 루틴이 끝날 때 까지 메인 루틴은 대기 상태에 있게 되며, 서브 루틴이 다 동작할 때 까지 어떤 작업도 실행하지 못한다.

![1000010161.png](1000010161.png)

- 반대로 비동기는 서브 루틴(코루틴)을 요청하여 실행하는 중에도 잠시 대기하고 메인 루틴을 실행할 수 있다. 이와 같이 동작하면 메인 루틴 중간에 코루틴의 필요한 작업만 호출하여 작업의 효율과 속도를 올릴 수 있다.

### 코루틴 구현 (라이브러리)

- 여기서는 파이썬으로 예제 코드를 확인하도록 하겠다.
  보통 파이썬으로 코루틴으로 비동기 작업을 하기 위해서는 `asyncio` 라이브러리를 사용한다.

```python
import asyncio # 코루틴 비동기 작업을 위한 asyncio 라이브러리 import

async def do_async(): # 함수 앞에 async 키워드를 사용하여 코루틴 생성
    pass
```

### 코루틴 구현 (제너레이터)

- 앞서 설명한 라이브러리를 사용하여 비동기 작업을 할 수 있는 코루틴 함수를 만들 수 있지만 다른 방식으로도 이를 구현할 수 있다.
- 하지만 이 방법은 파이썬만 가능한 방법이기에 주의해야 한다.
- 제너레이터를 사용하여 코루틴을 구현할 수 있다. 제너레이터는 메모리를 적게 사용하면서 높은 성능의 반복을 구현할 수 있다.

```python
def load_file(filename):
    purchase = []
    with open(filename) as f:
        for i in f:
            *_, price_raw = i.partition(",")
            purchase.append(float(price_raw))
    return purchase
```

- 위 코드는 제너레이터를 사용하지 않은 코드이다. 이 코드의 함수를 제너레이터 함수로 만들어 보자

```python
def load_file(filename):
    with open(filename) as f:
        for i in f:
            *_, price_raw = i.partition(",")
            yield float(price_raw)
```

- 위에서 볼 수 있듯이, yield 키워드를 이용하면 크기가 큰 리스트를 만들 필요없이 적은 메모리 사용량으로 기능을 구현할 수 있다.
- 다음으로 next, close, send 등의 함수를 사용하여 원할 때 동작을 호출하면 제너레이터로 코루틴을 만들 수 있다.
